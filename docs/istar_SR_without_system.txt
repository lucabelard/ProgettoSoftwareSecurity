## ACTOR: Mittente (Pharmaceutical Sender)

### Main Goal
**G1: Spedire Farmaci Mantenendo Catena del Freddo**

### Goal Decomposition (Task-Decomposition Links)
G1 is decomposed into:
- T1: Selezionare Corriere Certificato
- T2: Preparare Imballaggio Termico
- T3: Installare Sensori IoT
- T4: Depositare Fondi in Escrow
- G2: Verificare Conformità Post-Consegna
- G3: Pagare Solo se Conforme

### Subgoal G2: Verificare Conformità Post-Consegna
G2 is decomposed into:
- T5: Richiedere Report Sensori
- T6: Verificare Certificati Corriere
- T7: Ispezionare Pacco all'Arrivo
- T8: Analizzare Dati Temperatura

### Subgoal G3: Pagare Solo se Conforme
G3 is decomposed into:
- T9: Fornire Evidenze a Intermediario
- T10: Attendere Decisione Intermediario (MANUALE)
- T11: Contestare se Non Conforme

### Means-End Links
- T3 (Installare Sensori IoT) → R1 (Dati Oggettivi Condizioni)
- T8 (Analizzare Dati Temperatura) → G2 (Verificare Conformità)
- T4 (Depositare Fondi in Escrow) → SG1 (Protezione Finanziaria)

### Softgoals
- SG1: Protezione Finanziaria
- SG2: Tracciabilità Completa
- SG3: Minimizzare Rischio Perdita Farmaci

### Contribution Links
- T3 (Installare Sensori IoT) → ++ → SG2 (Tracciabilità Completa)
- T4 (Depositare Fondi in Escrow) → ++ → SG1 (Protezione Finanziaria)
- T10 (Attendere Decisione Intermediario) → -- → SG4 (Efficienza Processo)
- T11 (Contestare se Non Conforme) → -- → SG5 (Costi Operativi)

### Problemi (Negative Softgoals)
- SG_P1: Dipendenza da Intermediario Centralizzato [problema]
- SG_P2: Fiducia Manuale in Corriere [problema]

## SUMMARY: Problemi del Sistema Tradizionale

### Mancanza di Trust
- I dati del sensore possono essere manomessi dal Corriere
- Nessuna garanzia crittografica di integrità
- Dati non immutabili

### Centralizzazione
- L'Intermediario è un single point of failure
- Rischio di bias, corruzione, errori umani
- Commissioni elevate

### Processo Manuale
- Validazione soggettiva delle evidenze
- Nessun calcolo probabilistico automatico (no Bayesian Network)
- Tempi lunghi per dispute e validazione

### Mancanza di Automazione
- Nessun pagamento condizionale automatico
- Tutti i processi richiedono intervento umano
- Possibilità di errori e ritardi
